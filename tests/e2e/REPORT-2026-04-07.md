### Root Cause Analysis of Agent Test Failures

The failures cascade from `test_agent_joins_irc`:

1. **First run**: E2E fixture wrote old-format config (`[irc]` section) → CLI rejected it.
   **Fix applied**: Rewrote `e2e_context` to use new flat config format + global `servers` config.

2. **Second run**: Config format fixed. `agent create` creates the tab correctly
   (verified via `zellij list-panes`), but Claude Code's
   `--dangerously-load-development-channels` prompt is not auto-confirmed.
   The `_auto_confirm_startup` patterns don't match the current prompt text.
   **This is a pre-existing issue, not caused by the tmux → zellij migration.**
